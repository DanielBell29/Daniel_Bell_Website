Scottish Heritage Site - Raspberry Pi Interactive Signage

While working at an AV company, I frequently heard from engineers about ongoing issues with one of our National Trust customers' interactive signage displays. They reported persistent problems with various components of the exhibit failing, which led to repeated service requests and disruptions.

The interactive signage is supposed to be a playlist that plays old Scottish songs from different genres.

After discussing the situation with the support engineer, I learned that the exhibit relied on an outdated PC running an old Flash application. This setup posed several challenges: the Flash application required a bulky PC that had to be housed in a server cabinet. From there, USB and HDMI transmitter/receiver kits connected the PC to the interactive display using two CAT6 cables for video and touch functionality. The complexity of this system, with numerous interconnected parts, made it prone to frequent failures.

Recognizing an opportunity for improvement, I proposed redesigning the Flash application as an HTML-based web page that could run locally on a Raspberry Pi installed behind the interactive display. The Raspberry Pi was configured to automatically open the web page in full-screen mode upon reboot, ensuring seamless operation even after power interruptions.

Here is a link to the completed web application. The new system was installed over a year ago and has been operating without a single reported issue. The customer has been so impressed with the solution that they have expressed interest in implementing similar systems across all their exhibits onsite.

Homepage

This page serves as the main interface, displaying all available options to the user. The four album covers allow users to open their respective playlists, while the buttons at the bottom provide an alternative way to navigate between genres. These buttons remain accessible even when a playlist is open, ensuring seamless browsing across genres.

Additionally, I implemented a "Pause All" feature for convenience and added user feedback for each genre selection, allowing users to easily identify the currently active genre.

To enhance usability, the system is configured to default back to this page if the display remains inactive for ten minutes, ensuring a consistent and user-friendly experience.

Book Sections

This is the landing page for each genre, designed to provide an immersive and visually engaging experience. The layout is styled to resemble an open book, adding a creative and thematic touch to the interface.

Users can navigate through the content by touching the left or right pages, allowing them to seamlessly move to the previous or next sections within the genre.

Book Open

Each song is accompanied by playback feedback displayed on the left side of the page, while a brief description or contextual information is presented on the right. This feature was an enhancement I introduced, as it was not part of the original design.

When users navigate between pages, the currently playing music continues uninterrupted until another song is selected. To prevent frustration among staff, no additional tracks play automatically once a song concludes. This ensures that users cannot simply select a song and walk away, leaving the system unattended.

All Songs

The final page provides a comprehensive view of the entire discography, organized by genre. This layout allows users to conveniently browse and access all songs from a single page.
